# Harness_Engineering 架构总结

## 整体架构概述
Harness_Engineering 是面向长周期自主任务执行的 **Profile 驱动多代理分层架构**，源自 Anthropic 的 Harness 设计思想，针对LLM执行长周期开发/问题解决任务设计，解决了长上下文管理、可扩展性、任务分阶段推进等核心问题。

整体分层架构如下：

---

### 1. 配置层 (`config.py`)
统一管理全局配置，包括：
- API 连接参数（兼容OpenAI协议，支持任何LLM服务商）
- Token 阈值（压缩阈值、重置阈值）
- 循环限制（最大Harness轮次、Agent迭代次数、错误阈值）
- 工作区路径和输出文件命名
支持环境变量和 `.env` 文件覆盖配置，适配不同部署环境。

---

### 2. 上下文生命周期管理层 (`context.py`)
解决长任务的两大核心问题：**上下文溢出** 和 **上下文焦虑（模型过早尝试结束任务）**，提供：
- 分词计数：兼容 `tiktoken` 精确计数，无依赖时自动退化为字符估算
- 上下文焦虑检测：基于规则识别最近消息中模型提前结束的信号
- 分级上下文压缩：按Agent角色保留不同比例的最近消息，总结旧上下文平衡连续性和上下文空间
- 全上下文重置：生成结构化检查点持久化到硬盘，用全新上下文窗口继续任务，彻底解决上下文焦虑

---

### 3. 单Agent核心层 (`agents.py`)
实现带工具调用的核心Agent循环，是所有角色Agent的基础：
- 原生支持并行工具调用，提升执行吞吐量
- 完善的错误处理和速率限制指数退避重试
- 结构化Trace记录所有执行事件到JSONL，便于调试和审计
- 内置修正机制：检测到模型"只描述计划不执行工具"时，自动 nudges 推动实际执行
- 兼容所有OpenAI兼容API，无需修改核心即可切换不同LLM

---

### 4. 渐进式技能管理层 (`skills.py`)
实现Anthropic提出的三级渐进式技能披露，避免上下文空间浪费：
- **Level 1**: 启动时仅扫描加载所有技能的元数据（名称+描述）注入系统提示
- **Level 2**: Agent自主判断技能相关性，按需读取完整技能文档
- **Level 3**: 技能文档引用的子参考文件也由Agent按需加载
核心设计：让Agent决定何时加载哪些技能，而非外部硬编码，最大化上下文利用率。

---

### 5. 可组合中间件层 (`middlewares.py`)
基于抽象基类的横切扩展机制，**无需修改核心Agent循环即可扩展能力**：
- 定义三个标准扩展钩子点：
  - `per_iteration`: 每轮迭代开始前执行
  - `post_tool`: 每个工具执行完成后执行
  - `pre_exit`: Agent准备停止前执行
- 中间件可自由组合，由Profile按需装配，内置循环检测（识别重复修改同一文件的死循环）、时间预算提醒等通用中间件。

---

### 6. 工具层 (`tools.py`)
统一实现各类Agent可调用工具，适配OpenAI函数调用协议，提供：文件读写、bash命令执行、git操作、端口转发、网页读取、开发服务器管理等常用开发/任务工具。

---

### 7. 核心调度层 (`harness.py`)
多角色Agent调度，实现标准的 `Plan → Build → Evaluate → Iterate` 循环，基于Profile适配不同任务场景：
- 支持多种开箱即用的Profile：`app-builder`（从需求构建Web应用）、`terminal`（终端CLI任务）、`swe-bench`（软件仓库issue修复）、`reasoning`（高知识量问答）
- 可选契约协商阶段：本轮开发目标由Contract Proposer提出，Contract Reviewer审核，达成共识后再开始构建，减少方向偏差
- 动态分数跟踪，达到合格阈值提前结束任务，避免不必要的迭代

---

### 8. 评估层 (`eval.py`)
批量任务基准测试的辅助入口，方便对框架做能力评估。

---

## 架构选择的代数结构
这个架构的核心设计哲学是**一切可组合**，本质选择了 **幺半群 (Monoid)** 代数结构：
1. 所有可扩展组件（中间件、工具、技能、Profile扩展钩子）都支持顺序组合操作
2. 组合操作满足结合律：`(A • B) • C = A • (B • C)`，不同的结合顺序不改变最终语义
3. 存在单位元：空扩展组件（不执行任何操作，不改变原有流程），满足 `A • 单位元 = 单位元 • A = A`

这种代数结构的选择给框架带来了极佳的可扩展性：开发者可以自由组合任意多个扩展组件，无需修改核心架构就能新增能力，完美契合Harness作为通用自主任务执行框架的定位。

---

## 使用方式
```bash
# 复制配置文件并填写API信息
cp .env.template .env

# 运行任务，默认使用app-builder profile
python harness.py "Build a pixel art game in the browser"

# 指定profile运行
python harness.py --profile terminal "Fix the broken git merge in /tmp/project"

# 查看所有可用profile
python harness.py --list-profiles
```
